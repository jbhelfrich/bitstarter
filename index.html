Blizzard offers one of the best APIs for website and application developers available today, with just about everything you could want available for free, reasonable traffic limits, and easy registration for large scale projects. There's even a massive list of api implementations for just about every possible programing language. So why on earth do we need one more?

I've been working on an idea for a Warcraft related website, and it struck me that MongoDB was a perfect match for the JSON data returned by the Blizzard API. And since I wanted to add a local cache of the chracter data (even Blizzard's servers can't be up all of the time) and to abstract things like checking Last Modified headers, embedding character data in user records, and the like, this project was born.
